# Workflow Levels — the intention-accreting stage map surrounding the build

*Architecture intention-point `cc.0.workflow-levels` (2026-07-10). Front: intent → prebuild. Back: build-output → verification → learning. The LLM-middle is bracketed OUT, deliberately.*

Source of record: `devlab/runtime/memory/architecture/cc.0.workflow-levels.20260710.133555068072.json`

---

## Summary

Unifies three existing records — the `dev-pipeline` forward device map, `D-dev-pipeline-stations` (post-build stations), and `feedback-edges` (the backward flow) — into **one explicit level map** in the napkin framing (`staged-intention-compiler-pipeline`, Akien 2026-07-10):

> Each **level** is a transform that takes the prior level's output + a knowledge body and **adds a layer** to a carried intent artifact — so intent flows deterministically from raw user input through prebuild, and from build-output through testing to long-term verification.

Each level carries **three edge types**: the forward build edge, the return-for-learning edge, and the return-for-higher-level-work edge.

Two properties beyond today's forward-only ticket chart:

1. **Bidirectional edges are first-class** — not a one-way chart.
2. **The pipeline is continuous** — the JSON ticket is *one station*, not the origin.

**Scope (Akien's explicit correction, 2026-07-10):** both halves *surrounding* the build. The LLM/inference build step in the **middle is out of scope for now** — solidify the intention-based scaffold on both sides first, decide the middle later.

**Cross-cutting invariant, threaded through every level:** *separation of powers* — no level grades its own output.

## Front half — intent → prebuild

### L1 · Interviewer (Moist von Lipwig — NET-NEW, not Vetinari)
- **Inputs:** `{user, history}`
- **Outputs:** intent artifact = intentions + whys + how-to-verify + constraints
- **→ forward:** L2 intention-extractor
- **↩ return-for-learning:** origin of the chain (no inbound learning edge; it is where learning ultimately lands as revised interview priors)
- **↩ return-for-higher:** n/a (top of stack)
- **Implementing:** **NET-NEW device** — the people-reader/intake role. Discworld codename **Moist von Lipwig** (does *not* exist yet as a device); functional class name **"Interviewer"** per `D-agent-subclass-and-de-discworld-2026-07-02` (this resolves that decision's open **B4**: the interviewer/intake role *is* in scope, built under its functional name). **Not Vetinari** — Vetinari is governance/admin (the return-for-higher review-surface owner), not intake.
- **Why:** intent must be captured as a durable artifact with its whys and its verification criteria at the source, or every downstream level is compiling guesses.

### L2 · Intention-extractor
- **Inputs:** `{intent artifact + relevant history}`
- **Outputs:** adds **hierarchical** intentions (present-tense contracts); the Intent Doc Hubert consumes
- **→ forward:** L3 designer
- **↩ return-for-learning:** ← L11 verification: "did this intention hold in reality?" revises the hierarchy
- **↩ return-for-higher:** ← escalate-to-design when a spec-shaped failure names an intention as the wrong artifact
- **Implementing:** `devices/intent/` (`IntentExtractorDevice`: device.py, store.py, tools.py) — partly exists; intentions store (intentions-as-living-entities); IntentionsOutline is Akien-authored source. "Intention extraction + subintention creation" = adding the hierarchical-intentions layer.
- **Why:** flat intent under-specifies; hierarchy is what lets a low-level failure resolve to the *right altitude* of intent to re-open.

### L3 · Designer (Hubert)
- **Inputs:** `{hierarchical intentions + design patterns}`
- **Outputs:** adds **architecture/design** → a decision (`D-*`)
- **→ forward:** L4 ticketer
- **↩ return-for-learning:** ← L11 verification: decision hypothesis confirmed/falsified feeds pattern reuse
- **↩ return-for-higher:** ← escalate-to-design (feedback-edges): a spec-shaped build failure resolves `produced_by` and **re-opens the producing decision** in the producer's review surface
- **Implementing:** `devices/hubert/` (device.py, tools.py, constraint_normalizer.py, repo_auditor.py); `/sorted` + `audit-design`
- **Why:** architecture is the layer that turns "what we intend" into "the shape that realizes it"; a decision is the durable, falsifiable unit a failure can re-open.

### L4 · Ticketer (Hubert emit + independent filing audits)
- **Inputs:** `{design/decision + ticketing rules + a prerun of prebuild steps (orientation, constraints)}`
- **Outputs:** writes the JSON ticket (intent, constraints, test/observability plan)
- **→ forward:** L5 prebuild
- **↩ return-for-learning:** audit run-records accumulate (audit-audits) → promote recurring smells to rules
- **↩ return-for-higher:** audit AMEND/SPLIT/DISCARD bounces the ticket back to L3/L2 (mis-scoped → triage + design Qs = "kick back to design")
- **Implementing:** `hubert/tools.py`, `cc_queue.py` add-path, examiner-station (audits run by an instance that did **not** draft the artifact), audit-hypothesis/design/ticket
- **Why:** the ticket is where intent becomes buildable; filing audits by an **independent** examiner are separation-of-powers at filing time — the author cannot grade its own narrative (gate-attack G2/G3/G6a).

### L5 · Prebuild (deterministic build_packet compiler) — **LIVE**
- **Inputs:** `{ticket}`
- **Outputs:** `build_packet.v1` — intent, hard_constraints, scored context_shortlist, sufficiency_gate, baseline-vs-helper token_measurement, determinism fingerprint; + constraint decoration; + prospective-preregistration firewall fire
- **→ forward:** `[BUILD — OUT OF SCOPE]`
- **↩ return-for-learning:** sufficiency_gate + prereg measure orientation-token savings (~51% Delta) and preregister the intended proof
- **↩ return-for-higher:** sufficiency_gate FAIL surfaces an under-specified ticket back to L4
- **Implementing:** `build_packet.py`, `sprint_preflight_brief.py`, `pre_inference_assemble.py`, `prereg.py`, hubert `constraint_decorator`; fires in sprint preflight. **Status: LIVE but surface-don't-block v1; a manual sprint can still skip it.**
- **Why:** this is the last deterministic layer before the LLM — it packages everything the middle needs so the expensive resolver spends tokens on the novel, not on re-orienting.

## Back half — build-output → verification → learning

### L6 · Test (independent tester)
- **Inputs:** `{built branch/diff}` → **Outputs:** test verdict → **→ forward:** L7 examine
- **↩ return-for-learning:** flake/coverage signals feed test-factory horizon
- **↩ return-for-higher:** red tests bounce to the build step (and, if spec-shaped, onward to L3/L4)
- **Implementing:** `devices/tester` (T-tester-rackmount, in flight), evaluator. **Invariant: the tester is not the builder.**
- **Why:** a builder grading its own tests is the self-grading-hollow hazard; an independent tester removes it without a human in the loop.

### L7 · Examine (independent examiner)
- **Inputs:** `{branch + audit checklist + minimal context}` → **Outputs:** audit verdict (smell/debris/precode) → **→ forward:** L8 prove
- **↩ return-for-learning:** audit run-records → audit-audits → rule promotion / dead-check pruning
- **↩ return-for-higher:** AMEND bounces to the build step or back to L4/L3
- **Implementing:** examiner-station (design), audit-smell/debris/precode; execution leaves a run-record, not a free claim
- **Why:** presence-checks pass template-dressed vagueness when the author runs them; routing to an independent examiner is filing-and-build-time separation of powers.

### L8 · Prove (independent verifier)
- **Inputs:** `{branch + ticket}` → **Outputs:** a proof — a commit-bound red→green a hollow build could **not** pass (proof-on-close, CP1) → **→ forward:** L9 merge
- **↩ return-for-learning:** proof corpus grows → earns eventual gate removal (pipeline-trajectory staircase)
- **↩ return-for-higher:** no authentic red form ⇒ close **shipped-unproven** naming the missing proof-lever (the lever *is* the return-for-higher signal)
- **Implementing:** `proof_emitter.py`, `evaluator/loop.py` (judge panel), `proof_store.py`. **Invariant: the prover is not the builder.**
- **Why:** proof is how an answer earns the right to cache; the prover being independent is what makes the cached answer trustworthy.

### L9 · Merge
- **Inputs:** `{proven branch}` → **Outputs:** merged to main → **→ forward:** L10 index/savestate
- **↩ return-for-higher:** merge conflict/regression bounces to the build step
- **Implementing:** branching model (D-branching-model-2026-07-08); commit-to-main while just Akien + CC
- **Why:** merge is the commit that makes the proven answer part of the substrate.

### L10 · Index / savestate
- **Inputs:** `{merged commit}` → **Outputs:** code index updated; slate/session state flushed → **→ forward:** L11 verification
- **↩ return-for-learning:** the index + savestate are what a future session reads to avoid re-deriving
- **Implementing:** `devlab/claudecode/code_indexer.py`, `/savestate`, `/day-close` roll-up
- **Why:** an answer that is not findable is not cached; indexing + savestate keep answers reusable (the whole point of making cognition compile).

### L11 · Verification (consequence / outcome) — the return-for-learning trigger
- **Inputs:** `{closed decision + observable evidence}` → **Outputs:** hypothesis verdict: confirmed / falsified / needs-more-time → **→ forward:** L12 long-term verification
- **↩ return-for-learning:** **this is the primary return-for-learning edge** — the verdict flows back to L2/L3 (did the intention hold? did the decision's hypothesis confirm?), closing intention → why → observation
- **↩ return-for-higher:** a falsified hypothesis re-opens the producing decision (L3)
- **Implementing:** `/outcome`, `consequence_aging.py`, `completion_audit.py`, consequence-gate at filing (audit-ticket check 19)
- **Why:** a decision with no consequence check closes on paper only; verification is what makes the learning loop fire in reality (gate-attack G7).

### L12 · Long-term verification (validity conditions)
- **Inputs:** `{store entries + current world}` → **Outputs:** validity_conditions resolved; stale entries **flagged** (never deleted, CP2) → **→ forward:** loops to L1/L2 as revised priors
- **↩ return-for-learning:** the rot-*rate* control — a stale fact is caught by the sweep instead of silently misleading a future session (the IMAP-claim failure mode)
- **↩ return-for-higher:** a ≥2-sweep CURATE flag surfaces a fact for human review
- **Implementing:** `validity_sweep.py` (`--apply` at day-close), architecture/validity-conditions
- **Why:** answers rot; long-term verification is what keeps the compiled corpus honest over months instead of decaying into confident-but-wrong.

## Edge types (cross-cutting)

### The one backward edge — `produced_by`
Every emitted artifact carries `produced_by` — a single directed blame edge naming the artifact whose **content** caused this emission ("if this is wrong, what should be reviewed?"). Stamped at the chokepoints (memory_emit, cc_queue add, proof emitter). This is the substrate that makes **both** return-edge types resolvable to a specific level.
*Why: one uniform edge type is what makes one dispatch rule possible; per-artifact-type escalation paths were the special-case trap (homogeneity over special-case).*

### return-for-learning
The consequence/outcome + validity loop (L11, L12) feeding verdicts back to the intent/design levels (L2, L3). Answers "did the answer this level produced hold up?" Non-blaming — it is the observe → learn → improve loop running at the design level.
*Why: forward edges are answers; this edge is the evidence that turns a one-time answer into a durable, trusted cache entry (or retires it).*

### return-for-higher-level-work (escalate-to-design)
The **one escalation verb** (builder result envelope, `outcome=escalated`, enriched — payload-field, **no bus change**) + the **one dispatch rule** at Granny reconcile: capability-shaped failures ride the tier ladder first; spec-shaped failures (or ladder exhaustion) resolve `produced_by` and land in the **producer's review surface**, re-opening the producing decision/intention. A failure re-opening the question a level already answered (graph-tree telos).
*Why: like a compiler blaming the source line via provenance instead of reporting "backend failed"; this is what moves the cost of specifying intent up-front where it belongs.*

### Invariant — separation of powers (threads every level)
No level grades its own output. Build-time: builder never tests/proves its own diff (L6/L7/L8 independent). Filing-time: author never audits its own ticket/decision (L4 examiner-station). This is the single property that lets the routine compile away from human supervision.
*Why: it doesn't matter that a builder can't self-certify — it shouldn't; independent test + examine + prove removes the self-grading-hollow hazard without a human in the loop.*
