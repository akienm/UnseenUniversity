## Decision narrative
UU has been building Prescient Agents 2.0 without naming it. The core insight (from a parallel Claude conversation on the 1991 McGregor paper): a Prescient Agent is not trying to make decisions — it is trying to continuously improve the quality of future decisions. This is the distinction between automation (Observe→Reason→Act) and anticipation. UU's four-layer architecture maps directly to PA2.0: Observe (channel posts, tool signals) → Relationship Discovery (pattern tracker, outcome learner) → Future Modeling (advisor loop, Vetinari) → Human Decision Support (CC sessions with Akien). Alejandro's coding factory pattern identifies the missing Layer 3 device: Vetinari takes project briefs → generates tasks with implementation details + testing criteria → routes to worker tier → automated proof of life → CI → manual QA. The master goal G-factory-of-factories: the framework can build anything; Claude stays for design; the system continuously improves its own routing decisions.

## Hypothesis
All work streams route through tiered cheap workers; CC is reserved for design with Akien; the observe→discover loop surfaces routing improvements automatically.

% of sprint tickets completed by non-CC workers; OUTCOME_LEARNER_REPORT shows routing insight signal; CC sessions per week trend toward design-only.
